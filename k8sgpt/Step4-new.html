Step 4. 사용자의 손과 눈을 확장하는 도구 - Amazon Bedrock과 K8sGPT를 활용한 AI 기반 Kubernetes 진단 시스템 구축 -

왜 이 실험을 시작하게 되었는가?

EKS 클러스터를 운영하다 보면 익숙한 문제들이 반복됩니다.

예: Pending 상태 Pod, 연결되지 않은 Service, 바인딩 실패한 PVC 등.

매번 사람이 보던 문제를, 이제는 AI가 '보고 말해주는 구조'로 바꾸고 싶었습니다.

그 상상에서 실험은 시작됐고, 핵심은 이것이었습니다:

"명령어 없이도 클러스터 상태를 진단하고 Slack 에 결과를 보내자."

그리고 이 실험에서 AI 분석 엔진으로 사용한 것이 바로 Amazon Bedrock 입니다. Bedrock 은 AWS 가 제공하는 AI 모델 허브입니다. 단순히 API를 넘는 수준이 아니라, 운영 자동화에 특화된 "AI 내장 플랫폼"입니다. 이 실험에선 Claude 3.5 Sonnet 모델을 서버리스 방식으로 연동했습니다.

이벤트 트리거 기반으로 작동된다고 하나 확인해본 결과 수동으로 Operator POD를 restart 하거나 재생성시에만 리벤트가 트리거 되고 있습니다. cronjob을 활용하여 주기적으로 사용 가능한 방법으로 생각해야합니다.

K8sGPT는 Kubernetes 클러스터 내 리소스를 AI 기반으로 분석하여 문제를 진단하고, 자연어 형태로 그 결과를 제공하는 도구입니다. OpenAl, Amazon Bedrock, Ollama 등 다양한 AI 백엔드를 연동할 수 있으며, Pod Crash, PVC 오류, Ingress 문제 등 일반적인 Kubernetes 이상을 빠르게 원인 해결 방법을 설명해줍니다.

  • kubectl 처럼 CLI 방식으로 사용할 수 있고,
  • Helm 기반 Operator 형태로도 배포 가능하며,
  • Slack 과 연동하여 보고가 가능합니다.

K8sGPT Operator 는 2023년에 최초 오픈소스로 등장했으며, 활발한 커뮤니티와 빠른 업그레이드 주기를 바탕으로 꾸준히 개선되고 있습니다.

결과 확인 방법

여기까지 확인 되면 serverless 모델 사용 준비가 완료되었습니다. 최초 설치시 POD 진단이 처음 1회 자동으로 됩니다. 확인 방법은 2가지입니다.

1. 명령어로 이벤트 조회 후 describe 로 확인

"kubectl describe results defaultpendingpod -n k8sgpt-operator-system" 결과

K8sGPT 진단 결과 알림 예시

K8SGPT Operator 자동 재시작 CronJob 구성 (with RBAC)

목적

K8sGPT 의 Slack Sink는 Pod 재시작 시점에만 진단 메시지를 전송합니다.

하지만 bedrock-config Pod 만 재시작해도 트리거가 발생하지 않으며, 실제로는 operator pod 가 재시작되어야만 Slack 으로 메시지가 전달됩니다.

이를 자동화하기 위해 Operator Deployment를 주기적으로 Rolling Restart 하는 CronJob 을 설정합니다.

문제

  • operator pod 재시작 시 Slack 메시지 5회 이상 중복 전송됨
  • 따라서 의도적이고 정기적인 재시작이 필요하며, Slack sink 의 중복 전송 현상은 인지하고 제한해야 함

구성 소감

이번 K8sGPT Operator + Claude 3.5 연동 실험은 단순한 기술 연결을 넘어, AI 기반 Kubernetes 운영이 실제로 가능한가? 라는 질문에 대한 검증이었습니다.

처음엔 "이게 과연 잘 될까?" 하는 의심도 있었지만, Bedrock 기반의 Claude 3.5 Sonnet 모델이 실제 진단에 사용되고, 그 결과가 Slack 에 자연스럽게 전송되는 것을 확인하면서 비록 100% 자동은 아니지만 AI가 인프라 운영의 언어를 이해하고 행동하는 구조를 직접 체감할 수 있었습니다.

특히 다음과 같은 점에서 의미가 컸습니다:

  • 모델을 눈으로 보고 활성화한 뒤 실질적으로 동작하는 구조를 만들었다다는 점
  • 결과 리소스가 실제로 생성되고, 이를 describe 로 확인하는 과정에서 "AI가 분석했다"는 실감이 들었던 순간입니다.
